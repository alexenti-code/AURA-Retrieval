Преодоление автофагии в мультиагентных LLM-системах через иерархические антагонистические контуры
    Advanced Unified Retrieval Architecture · Июнь 2026

    Современные Large Language Models в сценариях долгосрочного автономного функционирования ограничены линейным ростом KV-кэша и затуханием контекста. Попытки интеграции систем долгосрочной памяти через автоматическую генерацию синтетических данных и самообучение приводят к автофагии (Autophagy) — модель, обучаясь на результатах собственной генерации, замыкается в «эхо-камере», что вызывает экспоненциальный рост энтропии и деградацию связности памяти уже на 3–4 итерации.

    Настоящая работа описывает архитектуру AURA (Advanced Unified Retrieval Architecture). Стабилизация системы достигается за счёт разделения когнитивных функций на иерархические антагонистические контуры с дискретным квантованием количества ролей (3 → 6 → 9).

    1. Математический базис памяти AURA

    Динамическая память ИИ-агента оперирует концепцией полураспада информации (halfLife) по областям видимости (scope) и многофакторным композитным скорингом при извлечении. Каждому кванту памяти Mi присваивается временной вес, зависящий от его системной категории.

    При обработке входящего запроса функция извлечения максимизирует композитный score:

    Score(Mi) = α · Sim(Q, Mi) + β · exp(−Δt / τscope) + γ · Imp(Mi)

    где:
    
      Sim(Q, Mi) — косинусное сходство эмбеддингов запроса Q и кванта памяти Mi
      Δt — время с момента фиксации или последнего вызова кванта
      τscope — константа затухания, определяемая слоем классификации намерений (Agentic Intent Classification). Для scope = user_profile τ → ∞; для scope = session_task τ → 0
      Imp(Mi) — статический вес фундаментальной важности информации
      α, β, γ — динамические веса, пересчитываемые классификатором под интент пользователя (сумма строго равна 1.0)
    

    2. Динамическая формализация функции полураспада τscope

    Константа затухания τscope не является статической величиной. Значение τscope рассчитывается динамически на основе тензора признаков, формируемого Ролью 5 (Синдикатором) и Ролью 3 (Валидатором):

    τscope = τ0 · ln(1 + Freq(S)) · σ(Imp(Mi) / H(S))

    где:
    
      τ0 — базовый шаг дискретизации времени жизни:
        
          τuser_profile → ∞ (инвариантные бизнес-правила)
          τoperational_workflow = 2.6 × 106 сек (~1 месяц)
          τsession_task = 3.6 × 103 сек (1 час)
        
      
      Freq(S) — частота обращения к данному scope за последние N транзакций
      H(S) — энтропия Шеннона для пространства S: H(S) = −Σ p(m) · log₂ p(m)
      σ(x) — логистическая сигмоида, нормирующая влияние статической важности
    

    3. Теория когнитивного квантования: Матрица Триад

    Количество ролей в AURA строго дискретно и кратно трём: 3 → 6 → 9. Каждая тройка формирует ортогональный контур контроля: Теза (Генерация) → Антитеза (Критика) → Синтез (Управление).

    Контур А: Оперирование данными (Data Layer)

    
      Роль 1: Исполнитель (Colleague) — горячий контур (online). Принимает запрос Q, инициирует cross-role recall, рассчитывает Score(Mi), выдаёт ответ.
      Роль 2: Эволюционер (Intern) — асинхронный фоновый контур (near-line). Генерирует пограничные запросы, выявляет логические противоречия.
      Роль 3: Валидатор (Teacher) — фильтрует галлюцинации Роли 2, размечает метаданными, присваивает Imp(Mi).
    

    Контур Б: Оперирование алгоритмом (Code Layer)

    
      Роль 4: Системный Инженер (Executor) — холодный контур (offline). Монополия на модификацию кода и мета-параметров (α, β, γ, размер чанков). Работает по триггеру от Роли 5.
      Роль 5: Синдикатор (Researcher) — мета-мониторинг здоровья системы. Строит динамический граф cross-role recall.
        
          Триггер 1: скользящее среднее валидности Роли 3 ниже 0.80 на 50 транзакциях
          Триггер 2: фиксация «зацикливания памяти» — ошибка калибровки τscope
        
      
      Роль 6: Ментор (Mentor) — детерминированный Guardrail. Проверяет патчи от Роли 4. Отклоняет при нарушении граничных условий (α + β + γ ≠ 1.0, шаг > Δmax = 15%).
    

    5. Аналитический разбор сходимости контуров

    Пусть Et — объём ложной информации в памяти на шаге t. В 3-ролевой системе оператор обновления памяти замкнут на себя:

    Et+1 = κ · Et + ξt

    где κ — коэффициент автофагии (κ > 1 при высокой интенсивности генерации синтетики), ξt — шум. При κ > 1 система экспоненциально деградирует (Et → ∞).

    В 6-ролевой архитектуре вводятся Роль 4 и Роль 6, изменяющие оператор RAG в зависимости от производной ошибки dE/dt:

    dE/dt = (κ − γt)E + ξdγt/dt = η · Sign(dE/dt) · I(E > Ethreshold)

    где γt — компенсирующий фактор, η — скорость адаптации (жёстко ограничена Ролью 6: η max). Поскольку Роль 6 через статический анализатор гарантирует, что γt не выйдет в область нестабильности, система сходится к устойчивому аттрактору, где dE/dt ≤ 0.

    6. Дискуссия: Почему AURA — это не Fine-Tuning

    Критически важным является проведение жёсткой границы между внутренней пластичностью нейросети и внешней системной эволюцией.

    Попытки реализовать непрерывное самообучение через дообучение весов базовой модели несостоятельны по трём причинам:
    
      Катастрофическое забывание: модель теряет базовые навыки при частых градиентных сдвигах
      Вычислительный паралич: постоянный расчёт градиентов для 70B+ моделей экономически невозможен
      Vendor lock-in: изменения в весах модели превращаются в мёртвый груз при смене поколения LLM
    

    Архитектура отчуждаемой памяти

    AURA полностью исключает языковую модель из процесса изменения внутренних весов. LLM рассматривается как stateless процессор (CPU), принимающий на вход очищенный и взвешенный контекст. Вся эволюция бизнес-логики, накопление опыта и очистка от шума происходят во внешней структуре — слое обвязки, независимом от модели.

    ┌────────────────────────────────────────────────────────────┐
│               ВНЕШНИЙ КОНТУР AURA                           │
│  1. Структурированная база знаний (факты, процессы)       │ ◄── накапливается годами
│  2. Динамические матрицы скоринга (α, β, γ)               │
└────────────────────────────────────────────────────────────┘
                         │
                  Контекст-Пакет
                         ▼
┌────────────────────────────────────────────────────────────┐
│              ВЗАИМОЗАМЕНЯЕМОЕ ВЫЧИСЛИТЕЛЬНОЕ ЯДРО          │
│   [Модель N] ──► [Модель N+1] ──► [AGI]                   │ ──► Ответ
└────────────────────────────────────────────────────────────┘

    При выходе на рынок более производительных моделей вычислительное ядро заменяется без остановки системы. Накопленная годами внешняя память AURA остаётся нетронутой.

from typing import Dict, List
from dataclasses import dataclass, asdict

logging.basicConfig(level=logging.INFO, format='%(asctime)s [%(levelname)s] %(message)s')
logger = logging.getLogger("AURA_Engine")

@dataclass
class TransactionLog:
    query: str; response: str
    validation_score: float
    scope: str

class AURAEnvironment:
    def __init__(self):
        self.hyperparams = {"alpha":0.5, "beta":0.3, "gamma":0.2, "chunk_size":512}
        self.archive: List[TransactionLog] = []
        self.lock = False

    def metric_trend(self) -> float:
        return (sum(tx.validation_score for tx in self.archive) / len(self.archive)
                if self.archive else 1.0)

                self.env.lock = True
                await self._cold_circuit()
